Haptic Construction Kit

We've created a Haptic Construction Kit!

For researchers or intrepid cyborgs. Mix and match the modules you need to build the haptic wearable of your choice.

Current

Originally developed for vibration in a belt form factor, current work is extending the device for more tactors, with more on at a time, to create a 2D display for a back or tabletop form factor!

Getting Started

Greetings traveler. Presumably you're working at the CUbiC Lab at ASU, or perhaps you're a curious internet human. Either way, welcome.

To build a CUbiC Haptic Device, you'll need a few things:

  • You'll need some tactors. We have a couple options at this point, a single tactor, or a strip of 8. You may need to program them with firmware.
  • You need a controller, probably just an arduino - the most compatible is the FIO v3 and the haptic firmware.
  • Wires. Sadly because of our size constrains and legacy we're using a JST part which can be difficult to crimp yourself. Part numbers can be found in any of the hardware repositories
  • You can communicate with the belt with just arduino (or any other serial terminal), but you'll want a 'driver.' We have a cross platform python implementation or in windows we have a .net implementation
  • Finally, you'll want a host program, and eventually want to change it around to make it your own. On python theres an example .py in the pyhaptic driver. For windows .net several program have been written including Haptikos and Wizardofoz
